# Kinetic Browser Extension

The Kinetic extension is a WebAssembly-powered browser plugin that enables trustless, peer-to-peer resolution of `.kin` domains in the browser.

It acts as an interceptor for the `.kin` Top Level Domain (TLD). When a user navigates to a `*.kin` domain, the extension redirects the request to a local Sandbox environment (`resolve.html`). This sandbox hosts a **Kinetic Wasm Light Client** which securely fetches the website content from the Kinetic Mesh network without relying on any centralized HTTP gateways or Root Certificate Authorities.

## Architecture

*   **Background Service Worker / Script:** Intercepts `*.kin` requests using Chrome's `declarativeNetRequest` (for Chrome) or WebRequest APIs (for Firefox) and redirects them to the resolver.
*   **Wasm Light Client:** A Rust-based WebAssembly node that connects directly to the Kinetic DHT (Distributed Hash Table). It hashes the domain name to find the routing peer, handshakes securely, and streams the webpage data back to the browser.
*   **Resolver UI:** A seamless loading screen that presents the fetched HTML inside a secure iframe, isolating the decentralized content from the extension's privileged context.
*   **Popup UI:** A simple dashboard to enable/disable the interception and check node status.

## Building the Extension

Make sure you have Node.js and `npm` installed. We use Vite to bundle the extension.

### 1. Install Dependencies
```bash
npm install
```

### 2. Build for Google Chrome / Brave / Edge / Opera
```bash
npm run build:chrome
# The output will be in the /dist folder
```

### 3. Build for Mozilla Firefox
Firefox requires a slightly different `manifest.json` structure (it uses `background.scripts` instead of `service_worker`).
```bash
npm run build:firefox
# The output will be in the /dist folder
```

## Loading the Extension (Development)

### Chrome / Brave
1. Navigate to `chrome://extensions/` (or `brave://extensions/`).
2. Turn on **Developer mode** in the top right.
3. Click **Load unpacked**.
4. Select the `dist` folder generated by the build step.

### Firefox
1. Navigate to `about:debugging#/runtime/this-firefox`.
2. Click **Load Temporary Add-on...**
3. Select the `manifest.json` file inside the `dist` folder.

## Testing

Run unit tests via Vitest:
```bash
npm run test
```
